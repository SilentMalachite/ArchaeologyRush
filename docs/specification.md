# ArchaeologyRush 仕様ページ

最終更新日: 2026-03-15
ステータス: 実装進行中

## 1. 目的

- このドキュメントは、ゲーム実装の基準となる仕様を定義する。
- 考古学ドメイン仕様はユーザー確定値を基準に実装する。

## 2. スコープ

- 対象: ArchaeologyRush のコアゲームループ、データモデル、UI要件、品質要件
- 非対象: 実装詳細コード、未合意の考古学ルール

## 3. 用語集

| 用語 | 定義 | 備考 |
| --- | --- | --- |
| 発掘ターン | ゲーム内の1日分に相当する行動単位 | 1ターンで3アクション実行 |
| 層（Layer） | 土の層区分。深さと年代仮説を持つ | 上層ほど新しい |
| 遺物（Artifact） | 発見・回収・記録の対象物 | 土器片、石器、骨片、人骨、住居跡など |
| 記録（Catalog） | 遺物や層情報を台帳化する処理 | 記録完了で評価加点 |

## 4. ゲーム目標

- プレイヤーの達成目標: 制限ターン内に重要遺物を3点以上、記録欠落なしで回収する。
- 失敗条件: ターン上限到達時に重要遺物が3点未満、または重大な記録漏れが2回以上。
- クリア条件: 目標遺物数達成かつ最終レポートの必須項目が100%入力済み。
- 評価指標（スコア/品質）: 回収点 + 記録品質点 - 層破壊ペナルティ。

## 5. ゲームループ（1ターン）

1. 事前情報確認: 現在層、残り行動回数、未記録遺物数を確認。
2. 行動選択（例: 掘る/記録/回収）: 1アクションごとに `dig` / `catalog` / `recover` を選択。
3. 結果反映（状態更新）: 層深度、遺物状態、行動回数、スコアを更新。
4. ログ出力: 行動内容と成否理由をターンログに追記。
5. ターン終了判定: 残り行動0でターン終了。ターン上限または目標達成でゲーム終了。

## 6. ルール仕様（ユーザー確定）

### 6.1 発掘ルール

- 掘削可能条件: 対象セルが未掘削かつ行動回数が残っていること。
- 掘削コスト（時間/体力/道具耐久など）: 1回掘削で行動1消費、道具耐久-1。
- 禁止行為/ペナルティ: 同一ターンに同一セルを3回以上掘削した場合、層破壊ペナルティ-5。

### 6.2 層位ルール

- 層の生成/構成方法: 各発掘区画に上層・中層・下層の3層を初期配置。
- 上下関係の制約: 原則として上層から順に掘削し、未開放の下位層は操作不可。
- 破壊・混入の扱い: 不適切な連続掘削で混入フラグを立て、該当層の評価を減点。

### 6.3 遺物ルール

- 種別一覧: `pottery_shard`, `stone_tool`, `bone_fragment`, `feature_mark`.
- 発見条件: 層と座標ごとの確率テーブルに基づき `dig` 成功時に抽選。
- 回収条件: 発見済みかつ記録必須項目のうち「位置」「深度」を入力済み。
- 保存状態/価値評価: `poor` / `fair` / `good` / `excellent` の4段階で点数倍率を適用。

### 6.4 記録・検証ルール

- 必須記録項目: 遺物ID、座標、深度、層ID、発見ターン、担当者メモ。
- 記録漏れ時の影響: 未記録でターン終了した遺物は一時保留となり回収不可。
- 検証フロー: `catalog` 実行時に必須項目チェック -> 不足表示 -> 再入力。

## 7. データ要件（実装前提）

### 7.1 エンティティ候補

- `Excavation`
- `Layer`
- `Artifact`
- `TurnLog`
- `SiteState`

### 7.2 保存要件

- 永続化対象: 発掘セッション、層状態、遺物、確定済み記録、終了ターン結果。
- 一時状態: 画面選択中セル、入力途中フォーム、未確定アクション。
- 監査ログ要件: すべての行動を時刻・入力値・結果とともに `TurnLog` に保存。

## 8. UI要件（LiveView）

- MVP画面一覧:
  - 現場ビュー: 層と座標のグリッド表示、掘削済み状態を色分け。
  - 操作パネル: 行動ボタン、対象選択、簡易ステータス。
  - ログパネル: ターン内の操作履歴とエラー理由。
  - 実装済み最小画面: `Demo.run/0` の出力を表示する LiveView 画面
- 必須操作:
  - `dig`: 指定セルの掘削を実行し、発見判定を返す。
  - `catalog`: 発見済み遺物の記録入力と検証を行う。
  - `recover`: 記録済み遺物をインベントリへ移す。
- 表示更新方式: サーバー状態を単一ソースとして同期

## 9. 実装済み範囲

- `SiteState` に `dig` / `catalog` / `recover` / `end_turn` を実装済み
- `Excavation` にユースケース API と `game_status/1` を実装済み
- `game_status/1` は `:in_progress` / `:won` / `{:lost, reason}` を返す
- `Demo.run/0` で progression / winning / losing case を確認可能
- LiveView 最小画面でデモ出力をブラウザ表示可能

## 10. 非機能要件

- テスト:
  - ルールごとに正常系/異常系のExUnitを作成
  - バグ修正時は回帰テストを追加
- 品質ゲート:
  - `mix format --check-formatted`
  - `mix test`
  - `mix quality`
- 性能目標: 1アクション処理を200ms以内（ローカル実行基準）で完了。
- セーブ互換性: マイナーバージョン間でセーブ読込互換を維持。
